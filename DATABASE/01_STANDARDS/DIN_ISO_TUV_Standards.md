## 🔬 ДЕТАЛЬНОЕ ОПИСАНИЕ

### 1. DIN SPEC 91426:2020-12
**"Anforderungen an videogestützte Personalauswahlmethoden (VMP)"**

**Область применения:**
- Видеоинтервью (асинхронные и синхронные)
- AI-оценка во время видео-собеседований
- Цифровые Assessment Centers

**Ключевые требования:**
- Диагностическая валидность метода оценки
- Предотвращение системной предвзятости (Bias Prevention)
- Надёжность ПО (Software Reliability ≥ 99.5% uptime)
- Прозрачность алгоритмов оценки
- Конфиденциальность данных кандидата

**Практическое значение для кандидата:**
```
✅ Если компания использует видео-ИИ для оценки:
   → Она обязана соответствовать DIN SPEC 91426
   → Вы вправе запросить методологию оценки
   → Система должна быть валидирована на bias
```

### 2. ISO 10667-1:2020 & ISO 10667-2:2020
**"Assessment service delivery — Procedures and methods to assess people in work and organizational settings"**

**Часть 1:** Требования к заказчику услуг оценки  
**Часть 2:** Требования к поставщику услуг оценки

**Ключевые принципы:**
- **Fairness:** Справедливость процедуры для всех кандидатов
- **Transparency:** Прозрачность критериев оценки
- **Scientific validity:** Научная обоснованность методов
- **Feedback:** Кандидат вправе получить обратную связь
- **Confidentiality:** Защита данных оценки

**Области применения:**
- Психологическое тестирование (MBTI, Big5)
- Технические тесты и coding challenges
- Assessment Centers
- Структурированные интервью

**Практическая тактика:**
```
📋 На этапе Assessment Center:
   → Компания по ISO 10667 обязана сообщить критерии
   → Вы вправе запросить score breakdown после оценки
   → При отказе без обоснования — нарушение стандарта
```

### 3. prEN 18286 — AI Quality Management System
**"Quality management system for AI" (разработан CEN/TÜV)**

**Назначение:** Гармонизированный стандарт, дополняющий EU AI Act  
**Связь с EU AI Act:** Выполнение prEN 18286 = presumption of conformity с Art. 17

**Ключевые элементы:**
- AI lifecycle management (от разработки до вывода из эксплуатации)
- Dokumentation требований к данным обучения
- Continuous monitoring и performance tracking
- Человеческий надзор (Human oversight protocols)
- CE-маркировка для сертифицированных TÜV систем

**Для кандидата:**
```
🔍 Проверить сертификацию ATS:
   → Спросить HR: "Ist Ihr ATS-System TÜV-zertifiziert?"
   → Сертифицированные системы обязаны раскрывать метрики
   → Несертифицированные системы = правовой риск для компании
```

---

### 4. ISO/IEC 42001:2023 — AIMS (AI Management System)
**Первый международный стандарт управления системами ИИ**

**Требования к организациям:**
- Политика ответственного использования AI
- Оценка рисков AI-систем
- Обучение персонала AI literacy (обязательно с 2026 по EU AI Act)
- Аудит и непрерывное улучшение

## 🏷️ TÜV Сертификация AI-систем в Германии

### Что такое TÜV AI certification:
TÜV SÜD, TÜV Rheinland и TÜV Nord предлагают сертификацию AI-систем:

| Сертификат | Описание |
|---|---|
| **TÜV AI Quality Mark** | Качество и надёжность AI-системы |
| **TÜV AI Bias Audit** | Проверка на дискриминационную предвзятость |
| **TÜV GDPR Compliance** | Соответствие требованиям DSGVO |

## 🎯 КЛЮЧЕВЫЕ ВЫВОДЫ ДЛЯ СТРАТЕГИИ ПОИСКА РАБОТЫ

1. **Требуйте transparency:** Компании со стандартизированными ATS обязаны раскрывать критерии
2. **Используйте ISO 10667:** После оценки вы вправе запросить обратную связь
3. **TÜV-сертификация = доверие:** Это сигнал качественного и честного рекрутинга
4. **Документируйте всё:** Храните переписку — basis для жалоб при нарушении стандартов
